How CloudThumb Works

A serverless approach to media processing.

AWS Services Used

Amazon S3 AWS Lambda Amazon SQS Amazon API Gateway CloudWatch IAM Roles

Workflow Execution

1. Ingestion

Users upload high-resolution images. The Flask backend streams these directly to an S3 "Uploads" bucket.

2. Decoupling with SQS

Instead of processing the image immediately (blocking the user), we drop a message into an SQS Queue. This ensures the system remains responsive even under heavy load.

3. Serverless Processing

An AWS Lambda function is triggered by SQS. It downloads the image, uses the Pillow library to resize it, and uploads it to the "Thumbnails" folder.

4. Async Notification

The status is updated, and the user can see their thumbnail in the gallery immediately after processing is complete.
